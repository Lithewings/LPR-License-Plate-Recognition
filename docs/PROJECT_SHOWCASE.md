# 项目展示文案

这份文档用于 GitHub 项目主页、简历项目经历、面试讲解和作品集介绍。

## 1. 一句话介绍

基于 OpenCV 与 TensorFlow/Keras 实现的端到端车牌识别系统，覆盖车牌定位、字符分割、CNN 字符分类、训练评估和结果可视化。

## 2. 项目简介

本项目面向中国蓝牌识别场景，使用 OpenCV 完成车牌区域定位和字符分割，使用 TensorFlow/Keras 训练字符分类模型，并通过训练曲线、混淆矩阵、类别性能图和结果汇总表评估模型效果。项目保留了完整推理链路和中间可视化结果，便于复现实验、排查问题和对外展示。

## 3. 核心亮点

- **完整链路**：不是单独的分类模型，而是从原始车辆图片到最终车牌字符串的完整识别流程。
- **算法组合合理**：定位阶段使用传统视觉方法快速筛选候选区域，识别阶段使用 CNN 做字符分类，兼顾可解释性和识别能力。
- **中间结果可追踪**：保存车牌定位、车牌裁剪、字符分割和模型评估结果，便于定位问题。
- **训练评估规范**：不仅输出准确率，还生成混淆矩阵、分类报告和类别级性能图。
- **适合继续工程化**：可以自然扩展为命令行工具、Web Demo、API 服务或移动端轻量模型。

## 4. 技术实现摘要

车牌定位部分先将原图转换到 HSV 色彩空间，通过蓝色阈值生成候选掩膜；随后使用形态学开闭运算消除噪声并连接字符区域，再根据轮廓面积、长宽比和像素密度筛选车牌候选框。字符分割部分对裁剪后的车牌进行灰度化、对比度增强、二值化和字符区域筛选，并将单字符规范化为模型输入尺寸。字符识别部分使用 Keras CNN 模型输出字符类别概率，最终拼接为车牌字符串。

## 6. 面试讲解顺序

1. 先讲目标：输入一张车辆图片，输出车牌字符串。
2. 再讲流程：定位、裁剪、分割、识别、评估。
3. 讲为什么用 HSV：蓝牌颜色特征明显，比直接在 RGB 中阈值更稳定。
4. 讲候选框筛选：面积过滤、长宽比过滤、像素密度过滤可以减少误检。
5. 讲字符模型：统一输入尺寸，使用 CNN 分类每个字符。
6. 讲评估方式：混淆矩阵帮助发现相似字符误判，类别性能图帮助定位弱类别。
7. 讲局限和改进：当前主要适配蓝牌，可扩展绿牌/黄牌、多角度校正、无 GUI 批处理和 Web/API 服务。

## 8. 当前局限

- 对蓝牌更友好，其他颜色车牌需要扩展阈值和规则。
- 对严重倾斜、遮挡、反光、夜间低照度图片的鲁棒性有限。
- 当前推理脚本包含 GUI 展示逻辑，服务器部署前建议增加 `--no-gui` 模式。
- 当前仍是实验型脚本结构，后续可拆分为标准 `src/` 包结构并增加 CLI/API。

## 9. 后续优化路线

- 增加命令行参数，支持批量识别和无界面运行。
- 支持多颜色车牌定位，包括绿牌、黄牌和白牌。
- 增加透视矫正，提高倾斜车牌的字符分割效果。
- 将模型导出为 ONNX 或 TensorFlow Lite。
- 使用 FastAPI 或 Streamlit 增加在线演示页面。
- 增加自动化测试和示例数据，提升项目复现体验。
